## 상황 설명

macOS(Mac mini)에서 Python 3 표준 라이브러리만으로 작성된 투자 대시보드 자동 갱신 스크립트를 실행하면 **HTTP 429 (Too Many Requests)** 오류가 계속 발생합니다. 해결 방법을 알려주세요.

## 프로젝트 파일 구조

```
동학개미지표2/
├── investment_dashboard.html   # 티스토리 블로그에 붙여넣는 단일 HTML 대시보드
│                               # 내부의 /*DK_DATA_START*/ ... /*DK_DATA_END*/ 사이
│                               # JSON 블록(DK_DATA)만 스크립트가 교체함
├── update_dashboard.py         # 데이터 자동 갱신 스크립트 (python3 update_dashboard.py 로 실행)
└── 티스토리_업로드_가이드.md    # 사용 설명서
```

## update_dashboard.py 동작 방식

- 표준 라이브러리만 사용 (urllib.request, http.cookiejar, csv, json, re)
- 무료 공개 API에서 값을 받아 HTML 안의 DK_DATA JSON 블록을 교체
- 갱신 항목: 자산 8종(S&P500·나스닥·코스피·비트코인·금·원/달러·달러인덱스·WTI), 공포·탐욕 지수 2종, 버핏 지표, VIX, 수익률 곡선, 하이일드 스프레드, 김치 프리미엄, 섹터 ETF 11종, 미·한 경제지표, FOMC/NFP 캘린더

### 이미 시도한 429 대응 (효과 없었음)

1. Yahoo: 크롬 User-Agent + fc.yahoo.com 쿠키 세션 + 요청 간 0.8초 간격 + query1→query2 재시도 → 여전히 429
2. Yahoo를 최후 폴백으로 강등하고 FRED·Stooq·CoinGecko·Frankfurter를 1순위로 교체 → 그래도 429 발생

## 질문

1. 위 무료 API들 중 어떤 곳이 429를 자주 내는지, 각각의 실제 rate limit과 차단 기준(IP 기반인지, UA 기반인지)을 알려주세요. 특히 CoinGecko 무료 API(분당 5~15회 제한)와 Stooq(일일 호출 제한)가 의심됩니다.
2. 429가 발생한 정확한 소스를 찾으려면 스크립트에서 어떻게 로깅하는 게 좋을까요?
3. 키 없이 쓸 수 있는 더 안정적인 대체 무료 API가 있다면 항목별로 추천해주세요.
4. 같은 IP에서 매일 1회 실행하는 수준인데도 차단된다면, 캐싱·재시도 백오프·요청 분산 등 표준 라이브러리만으로 가능한 회피 전략을 코드 예시와 함께 알려주세요.
