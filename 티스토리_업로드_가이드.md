# 투자 대시보드 — 티스토리 업로드 가이드

## ✅ 사전 체크

티스토리는 `<style>`과 `<script>` 둘 다 허용하지만, **반드시 "HTML 모드"로 붙여넣어야** 합니다. 기본 모드에 붙여넣으면 태그가 모두 텍스트로 변환되어 깨집니다.

---

## 📋 5단계 업로드 절차

### 1단계 — 글쓰기 화면 진입
티스토리 관리 페이지 → **글쓰기** 클릭

### 2단계 — 기본모드를 HTML 모드로 전환
글쓰기 화면 우측 상단 `기본모드 ▾` 드롭다운 클릭 → **HTML** 선택

> ⚠️ 한 번 HTML 모드로 전환한 글은 기본모드로 다시 전환하면 줄바꿈 등이 깨질 수 있으니, **처음부터 HTML 모드**로 시작하세요.

### 3단계 — HTML 파일 내용 복사
1. 작업 폴더의 `investment_dashboard.html` 파일을 메모장 / VS Code 등 텍스트 에디터로 열기
2. `Ctrl+A` → `Ctrl+C` 로 전체 복사

### 4단계 — 티스토리에 붙여넣기
HTML 편집창에 `Ctrl+V` 로 붙여넣기

### 5단계 — 미리보기 → 발행
1. 우측 상단 **미리보기** 클릭하여 디자인 확인
2. 모바일 미리보기도 함께 체크 (반응형 정상 동작 확인)
3. 카테고리·태그·썸네일 설정 후 **발행**

## 🔄 매일 업데이트하는 방법 — 자동화 완료 ✨

이제 수동 수정이 필요 없습니다. 터미널에서 한 줄만 실행하세요:

```bash
cd "본인 작업 폴더"
python3 update_dashboard.py
```

스크립트가 무료 공개 API(FRED, Stooq, CoinGecko, Frankfurter, CNN F&G, Alternative.me, OECD, Upbit, 연준 공식 캘린더)에서 최신 값을 받아 `investment_dashboard.html` 안의 `DK_DATA` 블록을 자동으로 교체합니다. Yahoo Finance는 429 차단이 잦아 **최후 폴백으로만** 사용하며, 항목마다 소스를 2~3개씩 순서대로 시도하므로 야후가 완전히 막혀도 전 항목이 갱신됩니다. 그 후 갱신된 HTML 전체를 복사해 티스토리 글에 붙여넣으면 끝.

```bash
python3 update_dashboard.py --dry   # 파일 수정 없이 결과 미리보기
```

### 자동으로 갱신되는 항목 — 전부입니다 (수동 항목 없음)
자산 8종(가격·등락·스파크라인), 주식/암호화폐 공포·탐욕(전일·1주·1개월·1년 포함), 버핏 지표(Wilshire5000÷GDP 자동 계산), VIX, 수익률 곡선(오늘+1개월 전), 하이일드 스프레드, 김치 프리미엄, 섹터 11종, 미국 경제지표 4종(기준금리·CPI·실업률·연준 B/S), **한국 기준금리(FRED/IMF)·한국 CPI(OECD)**, 경제 캘린더(**FOMC는 연준 공식 페이지 자동 스크랩** + 매월 첫 금요일 NFP 자동 생성 — 2027년 일정도 발표되면 자동 반영). 종합 시장 신호등 점수와 모든 해석 문구는 HTML 안 JS가 숫자로부터 자동 생성합니다.

참고: 한국 기준금리·CPI는 IMF/OECD 집계 경유라 발표 후 반영까지 1~2개월 시차가 있을 수 있습니다(값 자체는 정확). FOMC 스크랩이 실패하면 스크립트에 내장된 2026 일정으로 자동 대체됩니다.

### 🔄 완전 자동화 (강력 추천 — GitHub Pages & Actions) ✨

이 저장소를 GitHub에 업로드한 뒤 **GitHub Pages**와 **GitHub Actions**를 연동하면, 내 컴퓨터(맥미니)를 켜두지 않아도 매일 아침 자동으로 데이터가 갱신되고 티스토리 대시보드도 실시간으로 연동됩니다. (매번 수동 복사-붙여넣기 할 필요가 없어집니다.)

#### 1. GitHub 저장소 설정 (초기 1회)
1. [GitHub](https://github.com/)에 가입 후 Public 저장소를 생성하고 로컬 코드를 push합니다.
2. 저장소의 **Settings** → **Actions** → **General** → 맨 아래 **Workflow permissions**에서 **"Read and write permissions"**를 선택하고 **Save**를 누릅니다 (Actions가 갱신된 HTML을 커밋/푸시하기 위해 필수).
3. 저장소의 **Settings** → **Pages** → **Build and deployment**에서 Branch를 `main`, 폴더를 `/ (root)`로 지정한 뒤 **Save**를 누릅니다 (웹사이트 무료 호스팅 활성화).
4. 텔레그램 아침 알림을 계속 받으려면 저장소 **Settings** → **Secrets and variables** → **Actions**에 `New repository secret`으로 아래 환경변수들을 등록합니다:
   - `TELEGRAM_BOT_TOKEN`: 본인의 텔레그램 봇 토큰
   - `TELEGRAM_CHAT_ID`: 본인의 텔레그램 Chat ID
   - `DASHBOARD_URL`: `https://본인아이디.github.io/저장소이름/investment_dashboard.html`

#### 2. 티스토리 글 등록 (최초 1회만 등록하면 평생 자동 갱신)
티스토리 글쓰기 화면 우측 상단에서 **HTML 모드**로 변경한 뒤, 아래 코드를 붙여넣고 발행합니다.
```html
<iframe src="https://본인아이디.github.io/저장소이름/investment_dashboard.html" width="100%" height="2400px" style="border:none; overflow:hidden;" scrolling="no"></iframe>
```

### 💻 로컬 자동화 (선택 — 맥북/맥미니 전용)
GitHub을 통한 웹 호스팅을 쓰지 않고, 맥미니 로컬 파일만 매일 아침 07:50에 자동으로 업데이트하려면 crontab에 추가합니다:
```bash
50 7 * * * cd "/Users/kimsun/프로그래밍/동학개미 지표2/동학개미지표2" && /usr/bin/python3 update_dashboard.py
```

단, 이 방식은 갱신된 HTML 내용을 매번 수동으로 복사해서 티스토리에 덮어써 주어야 합니다.

## ⚠️ 주의사항

1. **티스토리 스킨 충돌**: 사용 중인 스킨이 `*` 셀렉터로 글로벌 스타일을 덮어쓰면 일부 디자인이 깨질 수 있습니다. 그럴 땐 모든 클래스명이 `.dk-`로 시작하므로, `<style>` 내부의 셀렉터를 `.dk-root .dk-XXX` 형태로 감싸도록 일괄 치환하면 됩니다 (현재 대부분 이미 그렇게 작성되어 있음).

2. **본문 폭 제한**: 티스토리 스킨에 따라 본문이 760~880px로 제한될 수 있는데, 대시보드는 `max-width:1180px`이지만 컨테이너가 좁아도 반응형으로 자연스럽게 줄어듭니다.

3. **첫 발행 후**: 발행 직후 한 번 실제 페이지로 열어보고 폰트 로딩(Google Fonts), 애니메이션, 우클릭 차단까지 모두 정상 동작하는지 확인하세요.
